Introducing EvoBench: Can AI Agents Build a Compiler?

We present EvoBench, the first serial agent benchmark based on a real-world compiler construction course. Unlike existing benchmarks that test isolated tasks, EvoBench evaluates an agent's ability to evolve — building knowledge from lexer to code generation in a strict sequential pipeline.

Key innovation: the Resurrection Mechanism. When an agent fails at Task N, we inject golden answers so we can still evaluate Tasks N+1 through N+k. This solves the cascading failure problem that plagues serial evaluation.

First Results: mimo-v2.5-pro vs deepseek-v4-pro on Compiler Construction

We ran our first evaluations using OpenHands SDK (a real agent framework with terminal, file editor, and task tracker tools). Both models successfully completed Task 0 (environment setup) but struggled significantly with Task 1 (lexical analysis) and beyond.

mimo-v2.5-pro: Pipeline score 34.7/100, 463K tokens, 768s
deepseek-v4-pro: Pipeline score 34.7/100, 615K tokens, 835s

The gap between Task 0 (100%) and Task 1 (~18%) reveals a massive capability cliff. Compiler construction requires understanding formal grammars, token specifications, and precise output formatting — skills that current LLMs haven't mastered.

Why Compiler Construction Is the Ultimate Agent Benchmark

Existing benchmarks like SWE-bench test isolated bug fixes. But real software engineering is sequential and cumulative. You can't write a parser without a working lexer. You can't optimize IR without generating it first.

Compiler construction is uniquely suited for agent evaluation because:

  • Strong causal dependencies between tasks
  • Objective, automated grading (does the compiled program produce correct output?)
  • Graduated difficulty (easy → medium → hard)
  • Real-world relevance (every programmer uses compilers)

Building Evo-CLI: One Command to Rule Them All

We designed Evo-CLI with a Harbor-like architecture: a unified AgentBackend interface that abstracts over different agent frameworks. Currently supported: OpenHands SDK, Claude Code, Codex CLI, Kimi CLI, and raw OpenAI API.

The key insight: we don't try to wrap each framework's API. Instead, we give each agent a workspace (the YatCC codebase) and a task prompt, then let it operate freely. The evaluator only needs to run cmake and parse score.json.
