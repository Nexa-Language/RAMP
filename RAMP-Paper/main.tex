\begin{abstract}
Large language model (LLM) agents are rapidly evolving from coding assistants into autonomous software engineering systems. \xwz{However, existing}{yet} evaluation methodologies remain \xwz{largely centered on}{rooted in} static, isolated, \xwz{and}{} short-horizon benchmarks that fail to capture the dynamic complexity of real-world production workflows. \xwz{As a result, benchmark performance often poorly reflects practical capability under realistic runtime environments invovling long execution chains, tool interactions, dependency management, and iterative feedback loops.}{}
We \xwz{thus}{} present \eva{} (Runtime Assessment of Models in Production), a production-grounded \xwz{assessing}{evaluation} framework built upon the \yatcc{} integrated platform\xwz{, which has been extensively used in practice}{}. \eva{} provides a unified \xwz{assessment infrastructure}{evaluation architecture} supporting heterogeneous LLM providers and agent SDKs through standardized API \xwz{interfaces and runtime orchestration.}{access,} \xwz{\name introduces}{} realistic \xwz{compiler construction workloads characterized by long-horizon execution,}{long-horizon compiler construction workloads with} strict serial dependencies, \xwz{iterative feedback cycles, and complex toolchain interactions}{}.
\xwz{In addition, \name incorporates}{} a \resurrection{} mechanism that decomposes cascading \xwz{workflow failures}{failure} into \xwz{recoverable}{} diagnostic \xwz{stages}{components}, \xwz{enabling fine-grained analysis of execution behaviors and intermediate recovery capability}{}. \xwz{The design further introduces}{and} utility-oriented multi-dimensional metrics \xwz{that jointly measure}{measuring} both outcome quality and process efficiency. Across 22 model \warn{configurations} and \warn{four} agent backends\zzc{}{, and \warn{two workload difficulty levels}}, we demonstrate that \warn{serial} \xwz{assessment}{evaluation} reveals failure patterns invisible to static benchmarks, that resurrection recovers substantial downstream diagnostic signal (median 73.65-point pipeline score recovery), and that process cost is decoupled from outcome ($R^2 = 0.007$), providing diagnostic rather than predictive value. \eva{} thus establishes a new paradigm for agent \warn{evaluation} that moves beyond synthetic benchmarks toward continuous, runtime-observable, production-grounded assessment.
\end{abstract}


\keywords{agent model \zzc{assessment}{evaluation}, production systems, compiler construction, serial dependency, resurrection mechanism}
%We evaluate 22 model configurations across four agent backends and two workload difficulty levels. Our results demonstrate that production-oriented serial assessment exposes failure patterns that remain largely invisible to conventional static benchmarks. We further show that the Resurrection mechanism substantially improves diagnostic observability, achieving a median pipeline score recovery of 73.65 points by recovering downstream execution signals from failed workflows. Moreover, we observe that process cost exhibits little correlation with final task outcome ($R^2 = 0.007$), suggesting that runtime efficiency metrics provide complementary diagnostic insight rather than predictive performance indicators. Overall, RAMP establishes a production-grounded evaluation paradigm for agentic systems that moves beyond synthetic benchmarks toward continuous, runtime-observable, and deployment-oriented assessment.
