\begin{abstract}
% ABSTRACT PLACEHOLDER — will be filled after draft completion
% Current skeleton version; final abstract must be 150-250 words per RESTRICTS.yaml

Existing agent benchmarks predominantly evaluate static, isolated, one-shot task completion, making it difficult to measure cross-task evolution, cascading failure vulnerability, and process cost structure in real engineering pipelines. We introduce \evobench{}, a serial coding benchmark grounded in compiler construction, where agents must progress from lexer to parser to IR generation, optimization, and backend code generation along a strict dependency chain. To prevent predecessor failures from masking downstream capabilities, we propose a \resurrection{} mechanism that injects verified intermediate artifacts after failure, decomposing ``predecessor-unreachable'' from ``downstream-incapable.'' Beyond final task scores, \evobench{} systematically records tokens, interaction turns, shell/build/test retries, and failure types, extending agent evaluation from outcome-only to outcome--process joint analysis. Across multiple models and agent scaffolds, we show that serial benchmarks significantly reorder static rankings, that resurrection recovers diagnostic signals obscured by cascading failure, and that process cost metrics explain effectiveness differences among similarly-scoring agents. \evobench{} thus provides a benchmark paradigm closer to real engineering workflows for studying evolution, self-repair, and long-horizon coding behavior.
\end{abstract}
