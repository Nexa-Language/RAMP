<背景介绍>
这是一个 agent 评测任务的项目：
- 测试 agent 在限额下完成特定任务的能力。限额包括时间、llm调用次数。
- agent backend 目前固定为 openhands 。模型则多种多样。
- 每个评测任务都会根据一个已有镜像来启动容器。
- @eval/container-runs/run-pipeline--minimax-m2.5-tasks-0-5 是目前程序中一次任务的输出目录的完成例子。
- 这个程序运行在宿主机（不是本机）中，但是有部分逻辑运行在容器中（我不确定，你需要从已有程序的逻辑中确定）。
- 一个 agent 评测任务称为 job ；一个 job 中 agent 需要完成的任务称为 task 。
- @src/README.md 记录了本项目支持的所有 API 。
</背景介绍>

<项目结构原则>
以 API 为项目核心骨架，力求使 main.py 的功能与逻辑在使用上清晰、明确。
@src/main.py 仅包含对各类 API 的调用。
@src/core 放置被 @src/main.py 调用的功能模块，模块的功能划分清晰。其中文件名有下划线前缀的文件是不直接被 @src/main.py 调用的功能模块。
@src/lib 放置@src/core 中跨功能模块能够被共享的功能或函数。
</项目结构原则>
